# Member 1 — Data & Risk Engine

## Files I built
- data/applications.json          -> 10 fake applications with criticality + distribution flag
- data/license_rules.json         -> 11 license compatibility rules
- data/vulnerability_db.json      -> 150 simulated CVEs (generated by generate_vulns.py)
- data/sbom_dependencies.csv      -> 500 dependency rows across 10 apps (generated by generate_sbom.py)
- data/dependency_labels.csv      -> ground truth labels for self-testing (generated by generate_labels.py)
- backend/risk_engine.py          -> MAIN LOGIC: scores every dependency for risk
- backend/evaluate.py             -> self-test script (precision/recall)

## How my scoring works
Each dependency gets a risk score based on:
1. Vulnerability (CVSS score + bonus if no patch / exploit known)
2. License conflict (GPL-style licenses in externally-distributed apps)
3. Unmaintained (no updates in 2+ years)
Score is multiplied by app business criticality (high=1.3x, medium=1.0x, low=0.8x)
